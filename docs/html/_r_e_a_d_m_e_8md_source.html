3 ## Instructions
4 
5 Produce a small working application for review by your prospective employer, the
6 details of the application are provided below.
7 
8 The aim for the test is to understand how you structure your solution, and how
9 well you manage to turn that solution into a working application.
10 
11 You need to:
12 
13  * Produce working Object Orientated, unit-tested code.
14  * Provide instructions on running and building your application.
15  * Deliver the working application to your prospective employer.
16 
17 You are expected to complete the test alone without immediate help, however
18 should you have any questions or need more clarity then contact via Github
19 either by raising an issue or messaging directly.
20 
21 You are not expected to know all the technologies used in this exercise, however
22 you should be able to find information on the internet.
23 
24 There is no given time limit, however the test is designed to be short. Please
25 read the entire brief and then contact the person who gave you this assignment
26 if you feel you are unable to complete the task in the allotted time frame.
27 
28 ## The Assignment
29 
30 Before you begin to write your solution, you should first fork this repository.
31 Provide a link to your forked repository to the person who gave you this
32 exercise. You should (as with any project) commit and push regularly to your
33 forked repository.
34 
35 The solution should be written in C++.
36 
37 A boilerplate project has been provided as part of the repository. Candidates
38 are encouraged to use the provided framework as the basis of their project.
39 
40 The boilerplate project makes use of and uses the following dependencies:
41 
42  * CMake
43  * googletest
44  * git
45 
46 A `Vagrantfile` has been provided, it can optionally be used to build the
47 project using Centos 6.
48 
49 If any of the above software is unknown to you, then you are expected to
50 research the subjects yourself, factor this in to the estimate you give to
51 complete this task.
52 
53 The project has been designed with Cross-platform support in mind. Ensure that
54 your software builds and runs on at minimum `Centos 6`, and preferably
55 `Windows 2008 r2 or greater` and/or `Mac OS 10`.
56 
57 Efforts should be made maintain support for the following platforms:
58 
59  * Windows 2008 r2 or greater
60  * Centos 6 or other linux
61  * Mac OS 10
62 
63 We do not expect that you may have access to all of the OS platforms listed
64 above. Should you be unable to test on all of the above platforms make a note
65 of this fact here: MacOS
66 
67 The goal of this exercise is to produce a reusable library which provides basic
68 calculator functions.
69 
70 The calculator library should have methods for the following functionality:
71 
72  * Add a value
73  * Subtract a value
74  * Multiply by a value
75  * Divide by a value
76  * A method to return the total
77  * An undo method
78 
79 The library should have written tests, but also a simple console application
80 which demonstrates the library should also be provided.
81 
82 The application should:
83 
84  * Accept operations from stdin
85  * Print to stdout the running total
86  * a command to display the command history
87  * a command to undo
88  * a command to clear/restart
89 
90 Documentation on both the library and the console application should be
91 included.

93 ## Building the boilerplate project
94 
95 ### Linux / MacOS
96 
97 The following commands will build on linux:
98 
99 ```
100 mkdir build
101 cd build
102 cmake ../
103 make
104 make test
105 ```
106 
107 This project requires googletest, googletest can be installed system-wide or
108 locally, if installed locally set the `GTEST_ROOT` variable to the install
109 location of googletest.
110 
111 Note that the supplied Vagrantfile will build a linux VM and provision it for
112 building the project.
113 
114 ### Windows
115 
116 There are various ways to build on windows. This project has been tested using
117 MS Visual Studio Express 2010. Using other versions of Visual Studio and other
118 Generators should work fine as well.
119 
120 Download and install CMake make sure it is in your PATH.
121 
122 Download and unpack Google Test to `c:\googletest`
123 
124 Build googletest from the `Visual Studio Command Prompt (2010)`:
125 
126 ```
127 cd c:\googletest
128 cmake ./ -G "NMake Makefiles"
129 nmake
130 ```
131 
132 The CMake variable `GTEST_ROOT` can be configured should googletest be installed
133 in a different location.
134 
135 To build this project still from the `Visual Studio Command Prompt (2010)`:
136 
137 ```
138 cd c:<location of this repo>
139 mkdir build
140 cd build
141 cmake ../ -G "NMake Makefiles"
142 nmake
143 nmake test
144 ```
